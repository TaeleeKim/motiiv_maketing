# MOTIIV 마케팅 자동화 도구 프로젝트 결과 보고서

---

## 1. 주제 (프로젝트명)

**MOTIIV 마케팅 자동화 도구 (MOTIIV Marketing Automation Tool)**

건축·토목 엔지니어링 플랫폼 MOTIIV의 커뮤니티 마케팅을 지원하기 위한 반자동화 도구 개발 프로젝트

---

## 2. 목적

### 2.1 배경

건축·토목 엔지니어들은 "구조계산용 엑셀·스프레드시트"와 "실무 예제 아티클"을 온라인에서 적극적으로 검색하고 활용하는 특성이 있다. 이러한 특성을 활용하여 MOTIIV 플랫폼을 타겟 커뮤니티에 자연스럽게 노출하는 마케팅 전략이 필요하다.

### 2.2 프로젝트 목적

| 구분 | 내용 |
|------|------|
| **핵심 목표** | URL 콘텐츠를 분석하고, 관련 키워드를 추출하여, 댓글을 달 수 있는 커뮤니티 페이지를 자동으로 추천 |
| **워크플로우** | 완전 자동화가 아닌 **반자동화** 방식 채택 (링크/초안 추천까지 자동, 실제 댓글 작성은 사람이 직접) |
| **대상 플랫폼** | Reddit, Quora, Stack Overflow, 네이버 카페, 클리앙, 디시인사이드 등 국내외 커뮤니티 |

### 2.3 개발 범위

```
[입력] URL → [처리] 요약 + 키워드 추출 + 관련 페이지 검색 → [출력] 댓글 초안 + 추천 페이지 목록
```

## 3. 기대효과

### 3.1 마케팅 효율화

| 효과 | 설명 |
|------|------|
| **시간 절약** | 수동으로 키워드 분석 및 관련 게시물 검색에 소요되는 시간 대폭 단축 |
| **일관성 확보** | AI 기반 요약 및 키워드 추출로 분석 품질의 일관성 유지 |
| **채널 발굴** | 자동 검색을 통해 새로운 마케팅 채널(커뮤니티, 블로그 등) 지속 발굴 |

### 3.2 성과 측정 체계 구축

- **UTM 파라미터 자동 생성**: 출처별 클릭 추적 가능
- **채널별 ROI 분석**: 어떤 커뮤니티가 효율적인지 데이터 기반 판단
- **Google Analytics / HubSpot 연동 가능**: 기존 마케팅 분석 도구와 통합

### 3.3 브랜드 신뢰도 보호

- **반자동화 설계**: 스팸/봇 정책 위반 위험 최소화
- **인간 검토 필수**: 부적절한 댓글 방지
- **기술 기여 90% + 브랜드 홍보 10%** 원칙 준수 가능

## 4. 진행 내용

### 4.1 기술 스택

| 분류 | 기술 |
|------|------|
| **Frontend** | Next.js 14, React, TypeScript, Tailwind CSS |
| **Backend** | Next.js API Routes |
| **AI/LLM** | Google Gemini 2.5 Flash (무료 티어 활용) |
| **검색 API** | Serper API (Google Search, 무료 플랜 2,500회/월) |
| **크롤링** | Puppeteer (JavaScript 렌더링 페이지 대응), Cheerio (정적 페이지) |

### 4.2 시스템 아키텍처

```
┌─────────────────────────────────────────────────────────────────────┐
│                        사용자 인터페이스                              │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐              │
│  │  URL 입력   │ → │  옵션 설정   │ → │  결과 표시  │              │
│  │ (UrlInput)  │    │ (언어/필터) │    │(ResultCard) │              │
│  └─────────────┘    └─────────────┘    └─────────────┘              │
└─────────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────────┐
│                        API Layer (/api/process)                     │
└─────────────────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        ▼                     ▼                     ▼
┌───────────────┐    ┌───────────────┐    ┌───────────────┐
│   Scraper     │    │   Gemini AI   │    │  Search API   │
│ (Puppeteer/   │    │   (분석/요약) │    │  (Serper)     │
│  Cheerio)     │    │               │    │               │
└───────────────┘    └───────────────┘    └───────────────┘
```

### 4.3 핵심 모듈 구현

#### 4.3.1 URL 크롤링 (`puppeteer-scraper.ts`)

```typescript
// 주요 기능
- Puppeteer 기반 JavaScript 렌더링 페이지 크롤링
- 불필요한 요소(nav, footer, ads 등) 자동 제거
- 본문 영역 자동 탐지 (article, main, .content 등)
- 최대 5,000자 제한으로 API 비용 최적화
```

#### 4.3.2 AI 콘텐츠 분석 (`gemini.ts`)

```typescript
// 출력 형식
{
  "summary": "3문장 요약",
  "keywords": ["키워드1", "키워드2", ...], // 5개
  "commentDraft": "자연스러운 댓글 초안"
}

// 프롬프트 가이드라인
- 타겟 독자 관점 (구조/토목 엔지니어)
- 언어 선택 (한국어/영어/혼합)
- 광고 문구 금지, 기술 기여 중심
```

#### 4.3.3 관련 페이지 검색 (`search.ts`, `search-filters.ts`)

```typescript
// 검색 필터 옵션
- community: Reddit, Quora, Stack Overflow, 네이버 카페, 클리앙 등
- blog: Medium, Velog, 티스토리, 브런치 등
- forum: 뽐뿌, 보배드림 등
- news: TechCrunch, ZDNet, 조선일보 등
- all: 전체 검색

// 필터링 로직
- PDF 파일 자동 제외
- 논문 페이지 자동 제외
- 최대 5개 결과 반환
```

#### 4.3.4 UTM 추적 링크 생성

```typescript
// UTM 파라미터 구조
https://motiiv.com?utm_source={domain}&utm_medium=comment&utm_campaign={keyword}

// 예시
https://motiiv.com?utm_source=reddit.com&utm_medium=comment&utm_campaign=keyword_RC_보_설계
```

### 4.4 사용자 인터페이스

| 구성요소 | 기능 |
|----------|------|
| **URL 입력 폼** | 여러 URL 동시 입력 지원 (줄바꿈 구분) |
| **옵션 설정** | 타겟 독자, 키워드 언어, 검색 대상 필터 설정 |
| **결과 카드** | 요약, 키워드 태그, 댓글 초안, 추천 페이지 목록 표시 |
| **복사 기능** | 댓글 초안 및 UTM 링크 원클릭 복사 |

### 5.3 운영 가이드라인

마케팅 담당자가 준수해야 할 핵심 원칙:

1. **댓글은 반드시 사람이 최종 검토 후 게시**
2. **질문에 답변하지 않고 MOTIIV만 홍보하는 댓글 금지**
3. **각 커뮤니티 규칙(홍보 허용 여부) 사전 확인 필수**
4. **기술 기여 90% + 브랜드 홍보 10% 비율 유지**
5. **계산/식/코드가 포함된 답변은 실제 엔지니어 검산 후 게시**

## 6. 최종 요약

### 6.1 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | MOTIIV 마케팅 자동화 도구 |
| **개발 기간** | - |
| **기술 스택** | Next.js 14, TypeScript, Gemini AI, Serper API, Puppeteer |
| **배포 방식** | 내부용 웹 애플리케이션 |

### 6.2 핵심 성과

```
┌────────────────────────────────────────────────────────────────┐
│  [입력]                                                        │
│  • 분석할 URL (여러 개 가능)                                    │
│  • 타겟 독자, 언어, 검색 필터 설정                               │
└────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌────────────────────────────────────────────────────────────────┐
│  [처리]                                                        │
│  1. URL 크롤링 → 본문 추출                                      │
│  2. Gemini AI → 요약 + 키워드 5개 + 댓글 초안                   │
│  3. Serper API → 관련 커뮤니티 페이지 5개 검색                  │
│  4. UTM 파라미터 자동 생성                                      │
└────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌────────────────────────────────────────────────────────────────┐
│  [출력]                                                        │
│  • 콘텐츠 요약 (3문장)                                          │
│  • 추출된 키워드 (5개)                                          │
│  • AI 생성 댓글 초안                                            │
│  • 추천 댓글 대상 페이지 목록 (UTM 링크 포함)                    │
└────────────────────────────────────────────────────────────────┘
```

### 6.3 활용 시나리오

1. **새 스프레드시트/아티클 발행 시**
   - URL 입력 → 키워드 확인 → 관련 커뮤니티 게시물에 댓글 작성

2. **주기적 마케팅 활동**
   - 기존 인기 콘텐츠 URL 재분석 → 새로운 댓글 대상 발굴

3. **채널 효율 분석**
   - UTM 파라미터 기반 클릭 추적 → 효율 높은 채널 집중

### 6.4 결론

본 프로젝트는 MOTIIV 플랫폼의 커뮤니티 마케팅을 효율화하기 위한 **반자동화 도구**를 성공적으로 구현하였다. 완전 자동화 대신 반자동화 방식을 채택함으로써 플랫폼 정책 위반 위험을 최소화하고, 브랜드 신뢰도를 보호하면서도 마케팅 담당자의 업무 효율을 높일 수 있다.

향후 성과 데이터가 축적되면, 효율이 검증된 채널 위주로 집중하고 미구현 기능(성과 대시보드, 히스토리 저장 등)을 추가 개발하여 도구의 완성도를 높일 계획이다.
